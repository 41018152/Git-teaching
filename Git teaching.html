Git CMD 操作完整教學

1. 安裝 Git

Git 是版本控制工具，必須先安裝才能使用。
請前往官方網站 https://git-scm.com/downloads，根據你的作業系統下載並安裝最新版本。
安裝完成後，可以打開「命令提示字元（CMD）」或「PowerShell」，輸入 git --version 來檢查是否安裝成功。 

git --version

若顯示版本號，表示安裝成功。

2. 設定 Git 使用者名稱與電子郵件（只需設定一次）

Git 需要知道你的使用者名稱和電子郵件，這些資訊會顯示在每次提交（commit）中。
請使用以下指令設定全域（Global）名稱和信箱，這表示未來所有專案都會使用這組資訊。 

git config --global user.name "你的GitHub使用者名稱"
git config --global user.email "你的電子郵件信箱"

請將雙引號內的內容換成你自己的資料。這只需執行一次，除非想更改。

3. 初始化 Git 儲存庫

在你的專案資料夾裡（例如桌面上的某個資料夾），按住 Shift 鍵並滑鼠右鍵點擊空白處，選「在此處開啟命令視窗」或直接在命令列切換到該資料夾。
執行以下指令，讓該資料夾變成 Git 儲存庫： 

git init

此時該資料夾會有一個隱藏的 .git 資料夾，Git 開始追蹤這個資料夾內的變更。

4. 新增所有檔案並提交第一次版本

將目前資料夾所有檔案加入暫存區，準備提交。
建議每次新增或修改後都先執行 git add . 把所有變更加入暫存區。
然後用 git commit 建立一個提交版本，會帶上一個描述訊息，方便未來回顧。 

git add .
git commit -m "首次提交"
git branch -M main

git branch -M main 是把主要分支改名為 main，這是 GitHub 預設的主要分支名稱。

5. 建立遠端儲存庫並連結（Remote Repository）

到 GitHub 網站 https://github.com 登入，點選右上角的「＋」→「New repository」新建一個空的儲存庫（Repository）。
取一個名字，例如 my-project，不用勾選初始化 README 或其他檔案，建立後會得到一個網址。
範例網址： 

https://github.com/你的帳號/my-project.git

回到本機專案，執行以下指令連結遠端儲存庫（URL 請換成你的）： 

git remote add origin https://github.com/你的帳號/my-project.git

如果提示 error: remote origin already exists.，代表這個專案已經有設定遠端，可以用下面指令移除後再新增：

git remote remove origin
git remote add origin https://github.com/你的帳號/my-project.git

6. 將本地提交推送到遠端儲存庫

執行以下指令，將本地的主要分支推送到 GitHub 的遠端儲存庫，首次推送需要加上 -u 參數設定追蹤： 

git push -u origin main

之後再推送時只需執行 git push 即可。

7. 後續更新檔案提交與推送

當你在專案資料夾內新增、修改、刪除檔案時，可以用以下流程提交並推送最新變更：

git add .
git commit -m "更新說明"
git push

- git add .：把所有變更加入暫存區
- git commit -m "訊息"：提交版本並加上描述訊息
- git push：推送更新到遠端儲存庫

8. 查看目前遠端儲存庫設定

若不確定目前連結的是哪個遠端儲存庫，可以用以下指令查看 URL：

git remote -v

會顯示遠端名稱（通常是 origin）和對應的網址。

9. 刪除已設定的遠端儲存庫（若需要重新設定）

如果想重新設定遠端儲存庫，可以先移除舊設定：

git remote remove origin

之後再用第 5 步驟的指令新增新的遠端儲存庫。

10. 其他常用小提示

  • 每次執行提交前，建議先用 git status 查看目前有哪些檔案變更。
  • 如果只想新增特定檔案，可以改用 git add 檔名。
  • 在命令列複製貼上指令，建議貼上後先確認再按 Enter，避免誤操作。
  • 要離開命令列視窗，輸入 exit 或直接關閉視窗即可。
