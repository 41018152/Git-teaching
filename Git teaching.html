用 CMD 操作 Git - 完整教學

以下是使用 Windows CMD（命令提示字元）操作 Git 的完整步驟，包含如何建立本地儲存庫、設定全域使用者名稱與信箱、連結 GitHub 遠端儲存庫、推送檔案，以及如何查看遠端網址等。

1. 進入你的專案資料夾

例如你的專案在 C:\Users\你的帳號\my-project，執行：

cd C:\Users\你的帳號\my-project

2. 初始化 Git 儲存庫

讓該資料夾成為 Git 本地儲存庫：

git init

你會看到類似 Initialized empty Git repository 的訊息。

3. 設定全域使用者名稱與信箱（只需執行一次）

Git 需要知道你的身份，請用你的 GitHub 帳號和信箱替換下面指令中的內容：

git config --global user.name "你的GitHub使用者名稱"
git config --global user.email "你的信箱@example.com"

4. 新增檔案到 Git 暫存區（Stage）

將所有檔案新增至暫存區：

git add .

5. 提交檔案（Commit）

寫一個描述本次提交的訊息：

git commit -m "第一次提交：新增專案初始檔案"

6. 連結 GitHub 遠端儲存庫

假設你已經在 GitHub 上建立一個空的遠端儲存庫，URL 類似：

https://github.com/你的帳號/my-project.git

使用以下指令把遠端儲存庫加入本地專案：

git remote add origin https://github.com/你的帳號/my-project.git

7. 推送檔案到 GitHub

將本地的檔案推送到遠端的 main 分支（或 master，依你的遠端設定）：

git push -u origin main

第一次推送時加上 -u 參數設定預設追蹤分支，之後只要用 git push 即可。

注意： 如果你的遠端分支不是 main，請改成相應的分支名稱（例如 master）。

8. 查看已設定的遠端網址

你可以用下面指令查看遠端儲存庫網址：

git remote -v

執行後會看到類似：

origin  https://github.com/你的帳號/my-project.git (fetch)
origin  https://github.com/你的帳號/my-project.git (push)

9. 其他常用指令

  • git status ：查看目前檔案狀態
  • git pull ：從遠端拉取最新更動
  • git log ：查看提交歷史

附註：如何建立 GitHub 遠端儲存庫

請至 GitHub 官方網站，登入你的帳號後，點選右上角的 ＋ > New repository，填寫儲存庫名稱並建立空的儲存庫即可。

小結

透過以上步驟，你可以用 CMD 完整操作 Git，將你的專案從本機上傳並管理於 GitHub。推送成功後，也可以在 GitHub 上看到你的檔案和歷史記錄。
